# Legacy Invoicing Bridge Notes

This document provides schema translation rules and details the sync job between legacy database systems and the new PostgreSQL database.

## System Boundary
The legacy invoicing system (written in COBOL and SQL Server 2008) operates on customer records in batch mode. The `Legacy Invoicing Bridge` maps legacy invoice structures into our current subscription billing database.

## Schema Translation Rules
- `LEGACY.tblInvoices.invId` -> `production.invoices.id` (prepended with `leg-`)
- `LEGACY.tblInvoices.invAmt` -> `production.invoices.amount_cents` (multiplied by 100 to convert decimal to integer cents)
- `LEGACY.tblInvoices.invStatus` mapping table:
  - `0` (Created) -> `draft`
  - `1` (Sent) -> `open`
  - `2` (Paid) -> `paid`
  - `3` (Voided) -> `void`

## Known Sync Quirks
The SQL Server database does not support timezone offsets natively. All timestamps are extracted as UTC+1 (Central European Time) and must be shifted manually by -1 hour during ingestion scripts.
If batch ingestion hangs, run the cleanup stored procedure: `sp_clear_invoice_locks`.
