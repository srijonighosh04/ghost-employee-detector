# Observability Onboarding Guide

This onboarding guide maps out standard metrics, dashboard locations, and alerting paths.

## Metrics Architecture
We collect metrics via OpenTelemetry agents running on Kubernetes nodes.
- **APM**: Distributed traces are sent to Datadog APM.
- **Metrics**: Time-series metrics are stored in Prometheus and visualized in Grafana.
- **Log Aggregation**: Application logs are ingested into OpenSearch (ELK cluster).

## Core Dashboards
- **Grafana Workspace**: `http://grafana.internal.nexusiq.com/` (Folder: Platform-Core)
- **Key Metrics Dashboard**: `Platform Core Metrics V4`
  - Monitors JVM CPU spikes, API latency histograms (p95, p99), and database connection pool usage.

## Troubleshooting Alerts
If a `DatadogAPMConnectionTimeout` or `PrometheusDataLoss` alert triggers:
1. Verify OpenTelemetry collector pod health: `kubectl get pods -n observability`.
2. Check network policies on the observability namespace.
3. Restart the collector service if scraping logs lag behind real-time by more than 10 minutes.
