# Warehouse ETL SOP

This Standard Operating Procedure outlines data warehouse orchestration, ETL scheduling, and schema staging.

## Staging & DB Architecture
Raw system events are loaded from Amazon S3 bucket staging directories into the Snowflake raw database:
- `RAW.EVENTS.PAGEVIEWS`
- `RAW.EVENTS.TRANSACTIONS`
- `RAW.EVENTS.USERS`

## dbt Transformation Pipeline
We use dbt (Data Build Tool) to model, clean, and materialize metrics. Transformation runs hourly on a Kubernetes cron script:
1. **Source Validation**: Checks source schema compatibility and row-count margins.
2. **Materialization**: Updates target analytical dimensions `ANALYTICS.PROD.USERS_DIM` and `ANALYTICS.PROD.REVENUE_FACT`.
3. **Data Quality Tests**: Asserts unique constraints and non-null states on keys.

## Escalation Path
If the transformation fails, an alert is dispatched to PagerDuty. The on-call data engineer must manually restart the Airflow DAG `dbt_hourly_run` after clearing stale S3 manifest files.
