# Subscription Billing Engine Architecture

This document details the Stripe-integrated billing logic, dunning workflows, and database schemas.

## Dunning Schedule
When a recurring payment fails:
- **Day 1**: Re-attempt transaction; mark subscription status as `past_due`.
- **Day 3**: Send billing update email notification.
- **Day 7**: Second transaction retry.
- **Day 14**: Third transaction retry; lock access to core features.
- **Day 21**: Cancel subscription and mark customer account as `unpaid`.

## Reconciliation Routines
Reconciliation runs daily at 04:00 UTC via a cron job on the billing worker cluster. It compares Stripe ledger entries with the PostgreSQL `billing_transactions` table to flag discrepancy anomalies.

## Proration & Rounding Overrides
We support custom proration overrides for 3 legacy markets (UK, Germany, and France) where mid-cycle adjustments require a 15-day minimum window instead of real-time calculations. Decimal rounding is set to bank-style rounding (half-to-even) for all multi-currency invoice calculations.
